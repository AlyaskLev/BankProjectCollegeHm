# Идея проекта BankProjectCollegeHm и краткое описание.

Данный проект является домашним заданием в рамках курса "Python разработчик". Он нужен для обслуживания банковских операций, есть функции маскировки, обработки и фильтрации банковских операций, также вспомогательные функции. 


## Установка и использование.

1. Клонируйте репозиторий
```
git clone https://github.com/AlyaskLev/BankProjectCollegeHm.git
```
2. Установите зависимости 
```
pip install -r requirements.txt
```


### Импорт модулей и вызов нужных функций

Чтобы импортировать модуль, например `processing` нужно вести: `import processing`. 
Если нужны конкретные функции из модуля: `from src.processing import filter_by_state, sort_by_date`
или `from src.masks import get_mask_account, get_mask_card_number`

## Модули и функции

### Модуль masks

В модуле находятся функции для работы с банковскими операциями:

#### get_mask_card_number:

- Проверка маскировки номеров карт (валидные длины от 4 до 16 цифр).

- Тесты на неправильные типы и недопустимую длину.

#### get_mask_account:

- Проверка маскировки номеров счетов.

- Генерация исключений при недопустимой длине входных данных.

Генерация исключений при недопустимой длине входных данных.
#### Примеры использования:
```
get_mask_card_number
7000792289606361     # входной аргумент
7000 79** **** 6361  # выход функции

get_mask_account
73654108430135874305  # входной аргумент
**4305  # выход функции
```

### Модуль widget 

В модуле находится функция для работы с банковскими операциями

#### mask_account_card:

- Проверяет корректную маскировку номеров карт и счетов.

- Используются параметризованные тесты и фикстуры для различных входных данных.

- Тестируется обработка некорректных и пустых строк.

#### get_date:

- Проверяется преобразование дат из формата YYYY-MM-DDTHH:MM:SS и YYYY-MM-DD в формат DD.MM.YYYY.

- Обрабатываются некорректные строки и исключения.

#### Примеры использования
```
# Пример для карты
Visa Platinum 7000792289606361  # входной аргумент
Visa Platinum 7000 79** **** 6361  # выход функции

# Пример для счета
Счет 73654108430135874305  # входной аргумент
Счет **4305  # выход функции

# Пример для get_date
2024-03-11T02:26:18.67140 # входной аргумент
11.03.2024 # выход функции
```

### Модуль processing

В модуле находятся функции для работы с банковскими операциями:

#### filter_by_state:

- Фильтрация по значению ключа state (учитывается регистр).

- Параметризованные тесты с разными состояниями.

#### sort_by_date:

- Проверка сортировки по дате в порядке убывания.

- Обработка записей с одинаковыми датами.

- Проверка реакции функции на некорректные форматы дат.

#### Примеры использования:
```
# Выход функции со статусом по умолчанию 'EXECUTED'
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}]

# Выход функции, если вторым аргументов передано 'CANCELED'
[{'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, {'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}]

# Выход функции (сортировка по убыванию, т. е. сначала самые последние операции)
[{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, {'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}, {'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, {'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}]
```

#### start htmlcov/index.html
